What changes and why it matters

1
Deal message redesign Biggest impact
Current plain text → rich format with price, discount %, expiry timer, platform badge and 3 action buttons. This alone triples click-through rate.
2
Onboarding flow New
New users pick categories and platform on first /start. Stops irrelevant deals reaching users → fewer unsubscribes.
3
Wishlist + /search New
Users save deals and search by product name. Keeps them inside the bot instead of going to Google.
4
Daily digest New
Top 5 deals sent at 9 AM every day via node-cron. Users who miss live alerts still get value. Increases retention.
5
SQLite on Railway Volume Free
All user prefs, wishlist, interactions stored in dealradar.db on the volume. No external DB needed. Everything persists across redeploys.
6
Deal score badge Easy win
Auto-calculate 🔥 HOT / ✅ Good / 👍 OK based on discount %. Helps users instantly know if a deal is worth clicking.
7
Quiet hours + /settings New
Users set their preferred times to receive alerts. Respects time zones. Reduces block/mute rate significantly.

Core features

Smart deal format Day 1
Image + title + price + MRP + discount % + savings + platform + expiry timer. Auto deal score (🔥/✅/👍). 3 action buttons + 3 reaction buttons on every post.
Category filter onboarding Day 1
On /start, user picks from: Mobiles, Laptops, TVs, Audio, Wearables, Gaming, Appliances, All. Stored in SQLite. Only matching deals get sent.
Platform filter Day 2
User chooses Amazon / Flipkart / Both. Inline keyboard toggle in /settings. Respects filter when broadcasting deals.
Wishlist (save + view) Week 1
/save button on each deal. /wishlist shows all saved with current prices. /wishlist clear to remove all. Max 20 items per user.
/search command Week 1
User types /search boAt headphones. Bot searches deals table in SQLite by keyword. Returns top 3 matching active deals with inline buy buttons.
Quiet hours Week 2
User sets Do Not Disturb from/to hour. Deals posted during quiet hours are queued and sent when DND ends. Set via /settings → quiet hours.
Daily digest Week 2
node-cron job at 9 AM IST. Picks top 5 deals from last 24h by interaction score. Sent to all users with digest_on = 1. Can toggle off in /settings.
Deal interactions analytics Week 2
Track every 👍 👎 click save expired-report per deal. Admin command /stats shows top performing deals, CTR, most saved. Helps you post better deals.
Expiry auto-remove Week 3
Cron job every 30 min. Deals past expires_at get marked inactive. Expired deals no longer show in /search or wishlist. Expired? button lets users flag manually.
Admin /postdeal command Week 3
You post deals via bot itself: /postdeal title | price | mrp | url | platform | category | expiry. Bot saves to SQLite and broadcasts to all matching users instantly.
Price track + alert Week 4
/track <url> — bot checks price every 6h using axios+cheerio. Sends alert when price drops. Stored in price_tracks table. Max 5 tracks per user (free tier).
Referral system Month 2
/refer generates unique invite link. Bot tracks who invited who. Top referrers get early access to flash deals or exclusive categories.

Full command list — user facing

/startOnboarding flow — pick categories, platform preference. Shows welcome message with quick setup.
/helpShows all available commands with short descriptions. Clean formatted list.
/search <query>Search active deals by keyword. Returns top 3 results with price and buy button.
/wishlistView all saved deals. Shows current price. Options to clear or refresh prices.
/track <url>Track a product URL for price drops. Bot checks every 6h and alerts on drop.
/settingsChange categories, platform filter, quiet hours, daily digest on/off. Inline keyboard UI.
/todaysShow today's top 5 deals on demand (same as daily digest, on request).
/mutePause all alerts. Quick mute without changing settings. /unmute to resume.
/referGet your personal invite link. Share with friends to unlock perks.
/categoriesQuick toggle your category subscriptions without going into full settings.

Admin-only commands (your chat_id only)

/postdealPost a new deal: /postdeal title | price | mrp | url | platform | category | hours_valid
/statsBot stats: total users, active users today, top clicked deals, avg CTR, wishlist count.
/broadcastSend a plain text message to all users (for announcements, bot updates).
/dealdone <id>Manually mark a deal as expired. Removes from all wishlists and search results.
/usersShow user count breakdown: total, active 7d, by category, by platform preference.
